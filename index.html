Overview

The setup-claude-project.sh script creates an optimized project structure for development with Claude Code. Version 3.0 is a streamlined evolution that removes redundancy while maintaining all the power of previous versions.

What This Script Does

  • Creates standardized project documentation
  • Sets up session continuity files for multi-session work
  • Configures MCP (Model Context Protocol) infrastructure
  • Establishes skill definitions for automated workflows
  • Creates CI/CD pipeline templates
  • Initializes git repository with comprehensive commit
Philosophy: v3.0 recognizes that Claude Code has built-in knowledge of subagents, MCP, and protocols. Your project files should contain only project-specific guidance, not general documentation.

Quick Start

Installation

# Download the script
curl -O https://raw.githubusercontent.com/s220284/claude-code-setup/main/setup-claude-project.sh

# Make it executable
chmod +x setup-claude-project.sh

# Run it
./setup-claude-project.sh MyProject "A web application for task management"

# Or run interactively
./setup-claude-project.sh

After Setup

# Navigate to your project
cd MyProject

# Start Claude Code
claude

# Claude automatically loads CLAUDE.md at session start
Pro Tip: Claude Code reads CLAUDE.md automatically. This file contains everything Claude needs to understand your project's conventions, tools, and workflows.

Files Created

v3.0 creates a streamlined set of files, each with a clear purpose:

Core Documentation

CLAUDE.md Required

Purpose: Main session reference - auto-loaded by Claude Code

Contains:

  • Critical warnings (shell line endings, cloud-first development)
  • Engineering requirements (tests, docs, commits)
  • Session continuity protocol
  • Code conventions
  • Available skills & plugins table
  • Quick commands reference

PROJECT_STATE.md Required

Purpose: Current system status snapshot

Tracks component health, production state, critical files, recent changes, and known issues. Update after completing features.

SESSION_LOG.md Required

Purpose: Append-only work history

Each session adds an entry documenting work completed, files modified, decisions made, and next steps. Primary continuity mechanism for multi-session work.

CONTINUATION_GUIDE.md

Purpose: Quick-start commands for resuming work

Contains copy-paste commands for common tasks when returning to the project.

Development Workflow

The Three Pillars

Every production code change MUST include ALL THREE:

1. Tests

  • pytest with >80% coverage
  • Test success AND failure paths
  • Mock external dependencies

2. Documentation

  • Docstrings with Args/Returns/Raises
  • Comments for complex logic only

3. Git Commits

  • Conventional commit format
  • Co-Authored-By footer
  • Descriptive messages
NO EXCEPTIONS. NO SHORTCUTS. This discipline is what makes AI-assisted development reliable and maintainable.

Session Protocol

At Start of Session

  1. Read PROJECT_STATE.md for current status
  2. Read SESSION_LOG.md for recent work and next steps
  3. Check git status

At End of Session

  1. Update SESSION_LOG.md with work completed
  2. Commit any uncommitted changes
  3. List next steps

Critical Warnings

Shell Script Line Endings

The Write tool outputs CRLF line endings which break shell scripts on macOS/Linux.

For ALL .sh files, IMMEDIATELY after writing:

sed -i '' 's/\r$//' script.sh && chmod +x script.sh

Cloud-First Development

Never hardcode local paths. Use relative paths or environment variables.

# WRONG
path = '/Users/username/projects/myapp'

# CORRECT
import os
path = os.environ.get('APP_ROOT', '/app')
